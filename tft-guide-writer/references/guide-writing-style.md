# TFT 攻略笔记写作风格

本文件只约束攻略笔记正文，不约束版本更新公告。目标是保留现有攻略的清晰排版，但正文要更像中文 TFT 玩家之间的实战口令：少介绍，多判断；少解释，多动作；少翻译腔，多玩家口语。

## 核心目标

- 正文要直接回答读者这一局怎么打：什么时候能玩、前期怎么过渡、中期几级 D、D 到什么质量、后期怎么补上限。
- 每段优先写动作链，而不是介绍链。先给时间点和操作，再补必要原因。
- 每个句子只承担一个动作或判断。少用长句把条件、原因、结果全部塞在一起。
- 能写成硬条件时，不写成软铺垫。

## 句式模板

优先使用这类实战句式：

- `X开局再玩。`
- `2阶段用A过渡，装备先给B打工。`
- `3-2上6小D，D到主C/主坦两星后，先攒钱到50。`
- `4-2上8大D，阵容全2后上9。`
- `A没来之前先用B，A来后换掉C。`
- `三星后上人口，补D开高羁绊。`

写运营节奏时，动作顺序通常是：时间点 -> 操作 -> 目标质量 -> 后续节奏。

例如：

```markdown
3-2上6小D，D到前排和主C两星后，先攒钱到50。4阶段上8补齐羁绊，全2后存钱上9。
```

## 术语口径

运营节奏里优先使用玩家口语：

- `上6`、`上8`、`上9`
- `D牌`、`小D`、`大D`、`慢D`
- `全2`、`存钱到50`、`追三`、`上人口`
- `胡`、`挂`、`补`、`打工`
- `主C`、`副C`、`主坦`

正式机制、表格字段或术语校验场景可以保留系统写法；正文运营判断里尽量不要写得像翻译稿。

常见替换：

- `升到 Lv6` -> `上6` 或 `6级`
- `在 Lv6 持续刷新` -> `6级慢D`
- `补到2星` -> `D到全2`、`主C两星`、`主坦两星`
- 运营动作里的 `升星` -> 写清楚 `两星` / `整体两星` / `追三` / `3星`；正式强化符文名如 `升星之运` 不改
- `回存` -> `先攒钱到50` 或 `存钱到50`
- `继续往9级运营` -> `存钱上9`、`稳血上9`、`划水上9`
- `强化符文` -> 正文可写 `海克斯`；标题或正式条目可写 `强化/海克斯`
- `AP装备` -> `法系装备`
- `HP低于40%` -> `血量低于40%`
- `强势局` -> `优势局`
- `弱势局` -> `劣势局`

## 压缩规则

少用介绍型开头：

- 少写：`这是一套以...为核心的阵容。`
- 改写：`重装战士过渡，提莫带乐芙兰装备打连胜；8级看血量决定小D还是上9。`

把软判断改成硬条件：

- `可以考虑往这套走` -> `可玩`
- `更适合已经能...的对局` -> `X开局再玩`
- `上限会明显抬高` -> `有转职上限高`
- `整体质量补到2星` -> `D到全2`

少用连续缓冲词：

- `可以考虑`
- `更适合`
- `整体上`
- `相对来说`
- `通常`
- `明显`
- `后续再逐步`
- `由于...因此...`

这些词不是完全禁用，但连续出现会让攻略变慢。能直接判断时，直接判断。

## 强调和排版

- 每段只加粗 1 到 2 个最关键对象。
- 普通英雄名、常规装备名、常规羁绊名不必每次加粗。
- 连续运营句里不要让每个名词都变成视觉重点。
- 关键动作优先用短句和 `<u>...</u>`，不要用大段解释堆强调。
